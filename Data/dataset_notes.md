# Заметки по датасетам — методика расчёта метрик

Общая методика (см. `required_data.md`): метрики 2–7 считаются на выборке
(`SAMPLE_SIZE`) через streaming, метрика 1 (количество примеров) — всегда
точная, по всему сплиту. Тяжёлые объекты (картинки, HTML) в память не
накапливаются, в цикле собираются только скалярные признаки.

Статус разбора:
- ✅ [webcode2m.ipynb](webcode2m.ipynb) — WebCode2M / Vision2UI
- ✅ [webui.ipynb](webui.ipynb) — WebUI
- ✅ [websight.ipynb](websight.ipynb) — WebUI
- ⏳ датасет 4 — собранный русский синтетический датасет

## Как считается метрика в каждом датасете

| # | Метрика | WebCode2M | WebUI |
|---|---------|-----------|-------|
| 1 | Количество примеров | сумма `num_rows` из footer'ов 2063 parquet-шардов train (захардкожена константой — пересчёт дорог) | `num_examples` из `dataset_info` через Hub API (датасет маленький — 29 409 в train, можно спрашивать точно каждый раз) |
| 2 | Количество языков | нативное поле `lang` | поля `lang` нет — датасет полностью английский (`card_data.language: en`); вместо разнообразия языков считаем разнообразие `source_name`/`framework`/`css_framework`/`component_type` |
| 3 | Средний размер HTML | длина `text` (HTML+CSS слиты вместе) в символах и байтах | `html`, `css`, `js` — отдельные поля, размер каждого считается отдельно |
| 4 | Среднее количество DOM-узлов | lxml по `text`: элементы + текстовые узлы | то же самое по `html` + сверка с готовым полем `element_count`, посчитанным авторами датасета |
| 5 | Размер скриншота | одно значение W×H на пример, разрешение свободное | 3 фиксированных viewport'а на `sample_id`: desktop 1280×720, tablet 768×1024, mobile 375×812 — усреднять по всем строкам без разреза по `viewport` некорректно |
| 6 | Среднее количество CSS правил | декларации `property: value` внутри `<style>` и `style=` атрибутов (CSS встроен в HTML) | CSS — отдельное поле; считаем и декларации (для сравнимости с WebCode2M), и отдельно блоки-правила `selector { ... }` (буквальное значение "правило") |
| 7 | Среднее количество источников | среднее число уникальных доменов, на которые ссылается HTML одного примера (`href`/`src`/`data-src`) | есть явное поле `source_name` (github/codepen/awwwards/...) — считаем глобальное число уникальных источников по датасету; домены внутри HTML оставлены как вторичная метрика для сравнимости |

## Особенности, которые нужно держать в голове

**WebCode2M**
- 2.5М строк train — полный проход дорог, все метрики кроме (1) считаются по случайной выборке (`SAMPLE_SIZE`, `.take(N)` в streaming-режиме).
- HTML и CSS не разделены: CSS живёт внутри `<style>` и `style=` атрибутов.

**WebUI**
- Устроен как 3 строки на один `sample_id` — по одной на каждый viewport (desktop/tablet/mobile). HTML/CSS могут повторяться между вариантами одного сэмпла, это нарушает независимость наблюдений при усреднении.
- Существенно меньше (29 409 train) — можно проходить бОльшую долю сплита или целиком за разумное время (~5 мин на 8000 строк с картинками через streaming).
- Есть готовые категориальные поля (`framework`, `css_framework`, `component_type`, `source_name`) и bounding boxes каждого DOM-элемента — не используется в текущих метриках, но может пригодиться позже.
